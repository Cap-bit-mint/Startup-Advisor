# 路演框架库

## 一、Pitch框架体系

### 1.1 AARRR框架 (海盗模型)

**适用场景**: 数据驱动的投资人

**结构**:
```
1. Acquisition 获取
   - 用户来源
   - 获客渠道
   - CAC数据

2. Activation 激活
   - 首次体验
   - 关键动作完成率
   - 激活率

3. Retention 留存
   - 留存曲线
   - 流失原因
   - 留存策略

4. Revenue 收入
   - 收入模式
   - LTV
   - 付费率

5. Referral 推荐
   - K-factor
   - NPS
   - 病毒机制
```

**适用赛道**: 消费互联网、SaaS

---

### 1.2 精益画布 (Lean Canvas)

**结构**:
```
┌─────────────────────────────────────────────────────────────┐
│ Problem (问题)          │ Solution (解决方案)              │
│ 1.                   1.                                   │
│ 2.                   2.                                   │
│ 3.                   3.                                   │
├─────────────────────────────────────────────────────────────┤
│ Key Metrics (关键指标)   │ Unique Value Proposition (独特价值)│
│ 1.                   [一句话核心价值主张]                    │
│ 2.                   [副标题: 补充说明]                    │
├─────────────────────────────────────────────────────────────┤
│ Unfair Advantage (护城河) │ Channels (渠道)                 │
│                       1.                                   │
│                       2.                                   │
├─────────────────────────────────────────────────────────────┤
│ Cost Structure (成本)    │ Revenue Streams (收入)            │
│ 1.                   1.                                   │
│ 2.                   2.                                   │
└─────────────────────────────────────────────────────────────┘
```

**适用赛道**: 早期项目、任何赛道

---

### 1.3 哈顿矩阵 (Hax Matrix)

**适用场景**: 技术驱动型项目

**四问**:
1. **What** - 我们在做什么?
2. **How** - 我们怎么做?
3. **Why** - 为什么重要?
4. **Who** - 谁会获益?

---

### 1.4 使命-愿景-价值框架

**适用场景**: 情怀型投资人

**结构**:
```
使命 (Mission): 解决什么问题
    ↓
愿景 (Vision): 世界因此如何不同
    ↓
价值 (Values): 我们坚持什么
    ↓
行动 (Actions): 我们正在做什么
```

---

### 1.5 竞争聚焦框架

**适用场景**: 红海市场、高度竞争

**结构**:
```
1. 竞争格局
   - 传统方案
   - 现有竞品
   - 新进入者

2. 差异化
   - 技术壁垒
   - 成本优势
   - 体验差异

3. 护城河
   - 网络效应
   - 数据积累
   - 生态锁定
```

## 二、赛道特定框架

### 2.1 SaaS Pitch框架

**投资者关注**:
- LTV/CAC
- 流失率
- NRR (净收入留存)
- 产品粘性

**推荐结构**:
```
1. [开场] 一个巨大的市场机会
2. [痛点] 现有方案的局限性
3. [解决方案] 我们的方法
4. [商业模式] 如何盈利
5. [进展] 关键指标 (ARR增长率、NRR)
6. [团队] 为什么是我们
7. [融资] 资金用途
```

**关键指标**:
- ARR > $1M 或 > 3x YoY
- 流失率 < 10% (最好 < 5%)
- NRR > 100% (最好 > 120%)
- CAC回收期 < 12个月

---

### 2.2 电商/D2C Pitch框架

**投资者关注**:
- 单位经济
- 品牌差异化
- 获客效率
- 复购率

**推荐结构**:
```
1. [开场] 品牌故事 + 市场规模
2. [痛点] 消费者的未被满足需求
3. [产品] 差异化定位
4. [单位经济] 成本结构 + 利润率
5. [增长] 获客策略 + 复购数据
6. [供应链] 壁垒来源
7. [融资需求] 资金用途
```

---

### 2.3 Deep Tech Pitch框架

**投资者关注**:
- 技术壁垒
- IP保护
- 商业化路径
- 应用场景

**推荐结构**:
```
1. [问题] 技术瓶颈在哪里
2. [技术] 我们的创新
3. [验证] 技术可行性证明
4. [应用] 目标市场
5. [壁垒] IP + 先发优势
6. [商业化] 盈利模式
7. [团队] 技术+商业组合
```

---

### 2.4 平台型Pitch框架

**投资者关注**:
- 网络效应
- 双边平衡
- 生态系统

**推荐结构**:
```
1. [开场] 平台机会
2. [需求侧] 需求方痛点
3. [供给侧] 供给方动机
4. [匹配机制] 如何撮合
5. [增长飞轮] 双方如何相互促进
6. [壁垒] 网络效应
7. [数据] 积累劣势
```

## 三、数据展示框架

### 3.1 进展曲线

**原则**:
- Y轴从0开始 (诚信)
- 标注关键拐点
- 趋势优于绝对值

**模板**:
```
Y
↑
│                          ●
│                      ●
│                 ●
│            ●
│       ●
│  ●
└──────────────────────→ X
  时间/里程碑
```

---

### 3.2 对比图

**使用场景**:
- 我们 vs 竞品
- 现在 vs 过去
- 目标 vs 实际

**注意事项**:
- 公平对比
- 数据来源明确
- 避免误导性图表

---

### 3.3 漏斗图

**适用场景**:
- 用户转化
- 销售漏斗
- 增长路径

**模板**:
```
流量 100%
   ↓ 50%
活跃 50%
   ↓ 40%
付费 20%
   ↓ 50%
复购 10%
```

## 四、叙事框架

### 4.1 英雄之旅 (Hero's Journey)

**结构**:
```
1. 世界: [描述现状]
2. 召唤: [发现问题]
3. 拒绝: [尝试现有方案，失败]
4. 导师: [我们出现]
5. 跨越: [我们的解决方案]
6. 试炼: [过程中的挑战]
7. 奖赏: [阶段性成果]
8. 归来: [愿景: 改变世界]
```

---

### 4.2 问题-解决方案-价值

**简单有力版**:
```
Problem: [1句话痛点]
Solution: [1句话方案]
Value: [1句话价值]
Evidence: [1个关键数据]
```

---

### 4.3 数据叙事

**原则**:
- 每个数据讲一个故事
- 对比优于绝对值
- 趋势优于快照

**模板**:
```
"我们发现 [洞察]，
导致 [影响]，
所以 [行动]，
结果是 [结果]。"
```

## 五、路演禁忌

### 5.1 必须避免

| ❌ 禁忌 | ✅ 正确 |
|----------|----------|
| "我们没有竞争对手" | "我们的差异化是..." |
| "市场是万亿级" | "我们的SAM是$XX" |
| "我们是下一个XX" | "我们的独特之处是..." |
| "技术细节太多" | "用类比讲故事" |
| "团队照片很丑" | "专业的团队照" |
| "PPT超过20页" | "重点突出，10-15页" |
| "念稿式演讲" | "眼神交流，故事化" |

### 5.2 常见错误

1. **技术细节过多**
   - 投资人不懂技术
   - 他们关心商业价值

2. **市场数据虚高**
   - 会被追问SAM/SOM
   - 诚实更可信

3. **忽视竞争**
   - 假装没有竞争
   - 展示你的优势

4. **数据堆砌**
   - 3个关键指标 > 20个
   - 讲故事的逻辑

5. **缺乏重点**
   - 什么都想讲
   - 突出1-2个核心点

## 六、成功案例结构分析

### 典型成功Pitch元素

| 元素 | 常见做法 |
|------|----------|
| 开场 | 讲一个用户故事 |
| 痛点 | 用数据量化问题 |
| 解决方案 | 现场演示/截图 |
| 市场 | 清晰的TAM/SAM/SOM |
| 进展 | 增长曲线 |
| 团队 | 关键人物背景 |
| 结尾 | 清晰的融资需求 |

| 轮次 | 页数建议 |
|------|----------|
| Pre-seed | 8-10页 |
| Seed | 10-15页 |
| A轮 | 12-18页 |
| B轮+ | 15-20页 |

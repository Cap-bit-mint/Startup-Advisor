## 二、阶段化危险信号

### 2.1 概念期危险信号 (Pre-Seed)

| 信号 | 描述 | 量化标准 | 严重程度 |
|------|------|----------|----------|
| 伪需求迹象 | 无人主动询问解决方案 | 0个付费意向 | 🚨 红色警报 |
| 过度技术导向 | 只谈��术不谈用户 | 技术讨论>80% | 🔴 高危 |
| 市场规模虚高 | TAM计算无数据支撑 | 估值基于TAM | 🟡 中危 |
| 无目标客户 | 无法描述前100客户 | 客户画像缺失 | 🔴 高危 |
| 竞品认知缺失 | 声称"无竞争对手" | 竞争分析=0 | 🔴 高危 |
| 团队能力单一 | 全是技术或全是商业 | 背景同质化 | 🟡 中危 |
| 股权分配均等 | 创始人平分股权 | 无差异系数 | 🔴 高危 |
| 退出机制缺失 | 股东协议不完善 | 协议缺失 | 🟡 中危 |

**概念期信号关联图**:
```
伪需求 → 方向模糊 → 资源浪费 → 资金耗尽
   ↓
市场验证缺失 → 融资困难 → 团队解散
```

### 2.2 种子期危险信号 (Seed)

| 信号 | 描述 | 量化标准 | 严重程度 |
|------|------|----------|----------|
| 用户验证不足 | 付费用户增长停滞 | 月增长<5% | 🔴 高危 |
| MVP留存率低 | 用户不回访 | 次日留存<20% | 🚨 红色警报 |
| 单元经济为负 | CAC > LTV | LTV/CAC<1 | 🚨 红色警报 |
| 创始人冲突 | 重大决策反复讨论 | 决策周期>4周 | 🚨 红色警报 |
| 关键岗位空缺 | 3个月未填补 | 空缺期>90天 | 🔴 高危 |
| 融资准备不足 | Runway<6个月无动作 | Runway<180天 | 🔴 高危 |
| 产品方向摇摆 | 核心功能频繁变更 | 方向变更多次 | 🟡 中危 |
| 行业认知不足 | 无法回答专业问题 | 认知评分<3/10 | 🟡 中危 |

**种子期关键窗口**:
```
第1-3月: 验证问题真实性 → 10个付费访谈

第7-12月: 验证商业模式 → PMF达成
```

## 七、危险信号应对指南

### 7.1 响应原则

| 原则 | 描述 | 执行要求 |
|------|------|----------|
| **早发现** | 建立监控体系 | 指标可视化，异常报警 |
| **快行动** | 缩短决策周期 | 决策周期 < 1周 |
| **狠止损** | 必要时壮士断腕 | 及时止血，避免更大损失 |
| **稳心态** | 避免恐慌决策 | 理性分析，系统应对 |
| **透明沟通** | 及时同步信息 | 团队/投资人保持透明 |

### 7.2 应对策略库

**策略1 - 止血优先**:
```
适用: 资源消耗过快
行动:
  1. 削减非必要开支
  2. 暂停扩张计划
  3. 聚焦核心业务
  4. 加速融资准备
```

**策略2 - 聚焦核心**:
```
适用: 方向模糊/资源分散
行动:
  1. 明确核心价值主张
  2. 砍掉非核心业务
  3. 团队聚焦核心产品
  4. 简化组织结构
```

**策略3 - 团队调整**:
```
适用: 团队能力/协作问题
行动:
  1. 诊断核心能力缺口
  2. 关键岗位优先招聘
  3. 优化激励体系
  4. 必要时调整人员
```

**策略4 - 战略转型**:
```
适用: 市场/竞争重大变化
行动:
  1. 重新评估市场机会
  2. 寻找差异化切入点
  3. 测试新方向
  4. 准备Pivot或退出
```

### 7.3 常见应对误区

| ❌ 误区 | ✅ 正确做法 |
|---------|-------------|
| 忽视早期信号 | 建立预警机制 |
| 寄希望于"下轮融资" | 立即止血 |
| 归咎于外部因素 | 先内部后外部 |
| 等待奇迹发生 | 主动干预 |
| 恐慌性决策 | 理性分析后行动 |
| 隐瞒问题 | 透明沟通 |
| 拖延决策 | 设定决策时限 |

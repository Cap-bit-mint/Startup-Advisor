# 决策矩阵

## 一、战略决策框架

### 1.1 To-Do/Not-To-Do Matrix

| 决策类型 | 评估维度 | 权重 | 决策标准 |
|----------|----------|------|----------|
| 新功能开发 | 用户需求强度 | 30% | Top 3 需求 |
| | 技术可行性 | 20% | 核心团队可实现 |
| | 市场差异化 | 25% | 竞品无/弱 |
| | 单位经济影响 | 25% | 正向贡献 |

**评分标准**: 1-5分，≥ 3.5 启动，< 2.5 放弃

### 1.2 市场进入决策

```
                    高差异化
                        │
           进入          │        主导
                        │
低市场 ───────────────────┼─────────────────── 高市场
吸引力  │                           │
                        │
           观察          │        跟进
                        │
                    低差异化
```

| 象限 | 策略 |
|------|------|
| 高吸引力 + 高差异化 | 全力进入 |
| 高吸引力 + 低差异化 | 选择性进入 / 合作 |
| 低吸引力 + 高差异化 | 保持存在 |
| 低吸引力 + 低差异化 | 退出 |

### 1.3 赛道选择决策

| 评估维度 | 问题清单 |
|----------|----------|
| 市场 | TAM > $1B? 增速 > 15%? 集中度低? |
| 竞争 | 有明显痛点? 存在空白? |
| 时机 | 基础设施成熟? 认知教育完成? |
| 团队 | 有行业积累? 能快速建立壁垒? |

**通过标准**: 4项中至少3项优秀

## 二、融资决策框架

### 2.1 融资时机评估

| 因素 | 好时机 | 坏时机 |
|------|--------|--------|
| 数据 | 连续增长，趋势明确 | 波动或下滑 |
| 市场 | 估值上升期 | 寒冬或动荡 |
| 竞争 | 领先优势 | 被追赶 |
| 资金 | Runway 12-18月 | < 6个月 |

### 2.2 估值决策

```
                        高估值
                            │
        拿钱占位            │       争取主导
                            │
低股份 ──────────────────────┼───────────────────── 高股份
比例   │                            │
                            │
        保命要紧            │       精选投资人
                            │
                        低估值
                    多股份
```

**决策原则**:
- 早期: 股份 > 估值
- 中后期: 估值 > 股份

### 2.3 投资人选择

| 考量因素 | 权重 | 评估问题 |
|----------|------|----------|
| 资源导入 | 35% | 能带来客户/人才/战略? |
| 品牌背书 | 25% | 能提升下轮估值? |
| 条款保护 | 20% | 是否有坑? |
| 沟通效率 | 20% | 能及时响应? |

## 三、产品决策框架

### 3.1 功能优先级

**RICE框架**:
```
Priority = (Reach × Impact × Confidence) / Effort
```

| 分数 | 优先级 | 行动 |
|------|--------|------|
| > 100 | P0 | 立即开发 |
| 50-100 | P1 | 本季度完成 |
| 20-50 | P2 | 下季度考虑 |
| < 20 | P3 | 暂缓 |

### 3.2 Build/Buy/Partner

| 选项 | 适用场景 | 风险 |
|------|----------|------|
| Build | 核心差异化 | 资源投入大 |
| Buy | 非核心，紧急 | 集成风险 |
| Partner | 互补能力 | 依赖关系 |

**决策标准**:
- 核心能力: 100% Build
- 非核心紧急: Buy
- 互补能力: Partner

### 3.3 定价决策

| 定价策略 | 适用场景 | 优缺点 |
|----------|----------|--------|
| 成本加成 | 竞争激烈 | 简单，但难差异化 |
| 价值定价 | 差异化强 | 利润高，但需教育 |
| 竞品对标 | 成熟市场 | 安全，但难突破 |
| 动态定价 | 平台型 | 灵活，但复杂 |

## 四、团队决策框架

### 4.1 招聘决策

**Hiring Funnel**:
```
申请 → 筛选 → 面试 → 评估 → 试用 → 转正
 100%    30%     10%     5%     80%    100%
```

| 岗位类型 | 评估重点 | 试用关键指标 |
|----------|----------|--------------|
| 销售 | 成交流程 | 3个月成单 |
| 研发 | 代码质量 | PR通过率 |
| 运营 | 执行效率 | KPI达成 |
| 管理 | 团队建设 | 离职率 |

### 4.2 关键决策权限

| 决策类型 | 创始人 | 高管 | 经理 |
|----------|--------|------|------|
| 战略方向 | ✅ | ❌ | ❌ |
| 年度预算 | ✅ | 建议 | ❌ |
| 招聘(高管) | ✅ | ❌ | ❌ |
| 招聘(一般) | 建议 | ✅ | ✅ |
| 产品功能 | 原则 | ✅ | ✅ |
| 日常运营 | ❌ | ✅ | ✅ |

### 4.3 股权激励

| 阶段 | 建议比例 | Vesting |
|------|----------|---------|
| 核心高管 | 0.5-2% | 4年 |
| 关键员工 | 0.1-0.5% | 4年 |

## 五、退出决策框架

### 5.1 并购评估

| 维度 | 评估要点 |
|------|----------|
| 战略价值 | 协同效应多大? |
| 价格 | vs 独立发展价值 |
| 独立性 | 团队保留条件 |
| 时机 | 最佳窗口期 |

### 5.2 继续运营评估

**季度复盘问题**:
1. 核心假设是否还成立?
2. 里程碑达成率?
3. 资金消耗速度?
4. 团队状态?
5. 市场变化?

**决策树**:
```
是否有清晰盈利路径?
├── 是 → 资源是否足够?
│        ├── 是 → 继续执行
│        └── 否 → 融资/收缩/退出
└── 否 → 6个月能验证?
         ├── 是 → 聚焦验证
         └── 否 → 转型/退出
```
